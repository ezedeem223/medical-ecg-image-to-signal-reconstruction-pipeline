# Pipeline Compatibility Reports

**Label:** `SYNTHETIC_COMPATIBILITY_ONLY`

This directory contains outputs generated by the pipeline compatibility tools.
All reports describe engineering readiness and synthetic compatibility only.

---

## Report Files

| File | Generated By | Description |
|------|--------------|-------------|
| `asset_readiness_report.json` | `inspect_asset_readiness.py` | JSON inventory of all required runtime assets |
| `asset_readiness_report.md` | `inspect_asset_readiness.py` | Markdown version of asset readiness report |
| `synthetic_runtime_input_manifest.json` | `prepare_synthetic_runtime_inputs.py` | Manifest of the synthetic runtime fixture |
| `synthetic_pipeline_smoke_report.md` | `run_synthetic_pipeline_smoke.py` | Smoke test narrative report |
| `synthetic_pipeline_compat_summary.json` | `run_synthetic_pipeline_smoke.py` | Machine-readable smoke test summary |
| `PIPELINE_COMPATIBILITY_REPORT.md` | `generate_pipeline_compat_report.py` | Consolidated compatibility report |
| `submission_conversion_report.md` | `convert_submission_to_waveforms.py` | Submission-to-waveform conversion result |
| `predicted_waveforms/` | `convert_submission_to_waveforms.py` | Per-lead waveform CSVs if conversion succeeded |

---

## Interpreting Reports

- `can_run_full_pipeline: false` means required model checkpoints or data files are absent.
- `SKIPPED_ASSETS_MISSING` means the smoke test did not attempt runtime execution.
- `SKIPPED_INPUT_CONTRACT_UNAVAILABLE` means parquet support is missing or fixture is incomplete.
- `RUNTIME_EXECUTION_FAILED` may indicate expected domain mismatch, not a pipeline bug.
- `CONVERTED` means submission IDs parsed successfully; leads were not fabricated.

All scores on synthetic data **must not** be cited as real ECG reconstruction performance.
